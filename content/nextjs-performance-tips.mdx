Performance is part of the experience. For a portfolio or a technical dashboard, the page should feel responsive before the user reads a single line.

# 1. Optimize Images

Always use `next/image`

```tsx
import Image from 'next/image';

export function Hero() {
  return (
    <Image
      src="/hero.jpg"
      alt="Hero image"
      width={1200}
      height={600}
      priority // Load immediately for LCP
      placeholder="blur"
      blurDataURL="data:image/jpeg;base64,..."
    />
  );
}
```


## 4. Implement Proper Caching

| Strategy | Use Case | TTL |
|----------|----------|-----|
| `force-cache` | Static data | Forever |
| `revalidate: 3600` | Semi-static | 1 hour |
| `revalidate: 60` | Frequently updated | 1 minute |
| `no-store` | Real-time data | Never cache |

## 7. Keep technical sections cheap to render

For a blog or portfolio, I try to keep the hero, metadata, and content lists server-rendered where possible so the app stays fast on slower connections.

## Wrap-up

The practical goal is not just a good Lighthouse score. It is a site that opens quickly, stays readable, and does not get in the way of the content.
